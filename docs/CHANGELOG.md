# Journal des versions — HookBudget

Développé de façon incrémentale par sprints (S1 → S15).

| Sprint | Contenu                                                                   |
| ------ | ------------------------------------------------------------------------- |
| S1     | Base technique (Next.js, Tailwind v4, shadcn/ui, structure)               |
| S2     | Authentification (Better Auth : e-mail/mot de passe, sessions, reset)     |
| S3     | Couche de données (10+ modèles Prisma, enums, index, seed catégories)     |
| S4     | Dashboard temps réel (lecture seule, graphiques Recharts, dark mode)      |
| S5     | Module Catégories (CRUD, archivage, icônes/couleurs)                      |
| S6     | Module Revenus (CRUD, filtres, stats, graphiques)                         |
| S7     | Module Dépenses (CRUD, mode de paiement, récurrence)                      |
| S8     | Moteur de Budgets (recalcul auto, anti-chevauchement)                     |
| S9     | Objectifs d'épargne (contributions, progression)                          |
| S10    | Notifications & automatisations (centre, préférences, anti-doublon)       |
| S11    | Rapports & analyses (exports CSV/XLSX/PDF, historique)                    |
| S12    | Centre de paramètres (profil, sécurité, devise, langue, import/export)    |
| S13    | Assistant financier IA (chat streaming, insights, prévisions, résumés)    |
| S14    | Back Office (RBAC, utilisateurs, plans, audit, feature flags, paramètres) |
| S15    | Préparation production (tests, sécurité, CI/CD, Docker, PWA, SEO, docs)   |

> Aucun paiement réel n'est implémenté : les plans et le MRR sont des
> placeholders préparant une future intégration de facturation.
